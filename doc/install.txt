ІНСТРУКЦІЯ ПО ВСТАНОВЛЕННЮ


1. СИСТЕМНІ ВИМОГИ

При разробці й тестуванні викоритовувалося:
-- Веб-сервер Apache 2.4.0
-- Модуль PHP 7.1
-- Сервер баз данних MySQL 5.5
Можлива нормальна работа додатку й на інших версіях Apache/PHP/MySQL.


2. УСТАНОВКА СЕРВЕРНОЇ ЧАСТИНИ ДОДАТКА

2.1. Налаштування сервера баз данних MySQL.
На сервері баз данных MySQL створити нову базу данніх й коритувача для доступу до неї.

2.1.1 Обрати значення параметрів підключения до бази.
(налаштування дивіться в файлі base.inc)
Хост MySQL -- localhost 
База данних -- phones
Пароль -- (ввести пароль)

2.1.2 Створити нову базу данних ("phones").
Команда SQL:
CREATE DATABASE phones;

2.1.3 Створити структуру таблиць бази данних заповнити тестовими даними.
Для цього необходімо выконати SQL-скрипт (знаходится в архиві - [install/schema.sql]).


2.2. Розміщення й налаштування.

2.2.1 Добути вміст каталогу [phones] из архіву [phones.zip] в обране місце на веб-сервері.


2.3. Загрузка демонстраційної бази данних й налаштування.

2.3.1 Перейти до інтерфейсу адміністратора. Наприклад, якщо додаток встановлено як окремий хост (см. п. 2.2.1) з ім'ям "phones", слід набрати [http://phones/admin/].
Як з'явиться сторінка реєстрації, вказати:
Логін -- admin
Пароль -- admin

2.3.2 Завантажитиь демонстрацийну версію бази данних й цитати (посилання "Загрузити файли"). Приклад файлів [phones.csv] и [quote.csv] можна знайти в архіві в каталозі [install].

2.3.3 Змінити пароль для доступа до інтерфейсу адміністратора (посилання "Змінити пароль").

2.3.4 Вийти з інтерфейсу адміністратора й перевірити роботу додатка (набрати в браузері адресу хоста, наприклад, [http://phones/]).


3. ЗАВАНТАЖЕННЯ РОБОЧІЙ БАЗИ ДАНИХ

Заключний етап - підготовка й завантаження робочій версії бази данних.

Насамперед необхідно зібрати відомості про абонентів місцевої телефонної мережі та подати їх в електронному (табличному) вигляді.

Далі, за допомогою табличного редактора слід перетворити зібрані відомості у файл формату CSV (Comma Separated Value = текст, розділений комами). Це текстовий файл, де кожен запис таблиці розташовується в окремому рядку, поля поділяються символом ";" (точка з комою), кількість і порядок проходження полів для кожного запису наведено нижче (див. також зразок файлу [phones.csv] в архіві в каталозі [install]):
	
1)  number -		Номер телефона внутрішній
2)  number1 -		Номер телефона міський
3)  birthday -		День нарождения
4)  surname -		Призвище 
5)  name -			Им'я 
6)  patronymic -	По батькові
7)  office - 		Назва організації
8)  branindex -	 	Код підрозділу
9)  branch - 		Назва підрозділу
10)  funindex - 		Код відділу - (індекс порядку розташування запису на екрані)
11)  func - 			Назва відділу
12)  town_pre - 		Населений пункт тип
13)  town - 			Населений пункт назван
14)  street_pre - 	Вулиця тип
15)  street - 		Вулиця назва
16) house - 		Будинок
17) flat - 			Квартира (кімната) (чи - пуста строка)
18)	note - 			Нотатки
19)	sortindex -  	Порядковий номер для переважного сортування у відділі	
20)	userMail - 		Внутрішня поштова адреса


Залишилося завантажити базу даних на сервері з отриманого файлу CSV:
-- Змінити ім'я файлу на [phones.csv].
-- розташувати вказаний файл у каталозі [phones/admin] на веб-сервері.
-- в інтерфейсі адміністратора перейти до сторінки "Завантаження файлів" та виконати відповідну команду.

Після розміщення та тестування програми на одному із серверів, можна повідомити користувачів про доступність довідника в локальній мережі.


4. СУПРОВІД ДОДАТКА

У програмі передбачена можливість зворотного зв'язку користувачів з адміністратором програми, тому адміністратор програми повинен періодично переглядати повідомлення, що надійшли, і видаляти які вже не потрібні.

Також адміністратору необхідно іноді оновлювати відомості у базі даних, для чого можна залучити користувачів програми або спеціалістів вузла зв'язку.
